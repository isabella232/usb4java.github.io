Release 1.0.0 - 2013-05-01

Type Changes By
Rewritten library to use libusb 1.0 as backend. kayahr
Made library loading more universal to support loading shared libraries for platforms not officially supported by usb4java. kayahr
Add native library for linux-arm platform (Raspberry Pi). kayahr
Native libraries are no longer extracted to a temporary directory when they are not in a JAR file. kayahr
Improve error handling during native lib loading so javax.usb no longer hides the important exceptions. kayahr
Fix logic error in AbstractIrpQueue#isBusy(). kayahr

Release 0.5.0 - 2013-02-15

Type Changes By
Refactored project structure back into a single project. Native libraries are now included in the main JAR file. kayahr

Release 0.4.0 - 2012-11-12

Type Changes By
Corrected endpoint index in JNI libraries. kayahr
Refactored loading of native libraries. The libraries are now a standard Java dependency and are automatically extracted during runtime. So no java.library.path or LD_LIBRARY_PATH tinkering needed anymore. Just put the two JARs into your class path and that's it. kayahr

Release 0.3.4 - 2011-10-31

Type Changes By
Improved error messages when loading the JNI library fails. kayahr

Release 0.3.3 - 2011-03-28

Type Changes By
Fixed library paths in dylibs so they can be placed in *.app/Contents/MacOS. kayahr

Release 0.3.2 - 2011-03-27

Type Changes By
Now trying to load the JNI library with the following library names: usb4java, usb4java32, libusb4java, libusb4java32. kayahr

Release 0.3.1 - 2011-03-05

Type Changes By
Fixed missing support for custom buffer offsets and lengths. kayahr

Release 0.3.0 - 2011-02-20

Type Changes By
Fixed problem with empty control requests on Mac OS X. kayahr
Implemented configuration to control the communication timeout. kayahr

Release 0.2.0 - 2011-02-11

Type Changes By
Implemented configuration to control the scan mode and to improve scan performance by filtering for specific vendor and product ids. kayahr

Release 0.1.1 - 2011-02-08

Type Changes By
libusb reports broken bus root devices when detaching USB devices and when user has no root permissions on linux. Fixed it by only trusting root devices which are also in the device list of a bus. kayahr
